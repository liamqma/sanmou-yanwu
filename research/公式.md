【伤害公式总骨架】

```
final =
    ( 固定值 + 固定系数 × (攻方主属性 − 守方对应防属性) / 守方对应防属性 )   ← 基础数值项
  × 战法伤害系数                                                              ← 即技能标注的百分比，普攻 = 100% 兵刃
  × ( 1 + Σ 攻方各项「造成伤害」 )                    ← 区间A 通用槽：所有「造成伤害」线性叠加，100% 封顶
  × ( 1 + Σ 攻方各项「造成X伤害」 )                   ← 区间A 专属槽：「造成兵刃/谋略/主动/追击/普攻伤害」与「对X类目标伤害」（每个子类——对武将 / 对统率低于自身 / 对带异常状态的敌军 等——各自独立成乘区），同一标签内线性叠加且 100% 封顶
  × ( 1 − Σ 守方各项「使敌方造成伤害降低」 )         ← 区间B 通用槽：所有「使敌方造成伤害降低」线性叠加，100% 封顶
  × ( 1 − Σ 守方各项「使敌方造成X伤害降低」 )        ← 区间B 专属槽：「使敌方造成兵刃/谋略/主动/追击/普攻/对X类目标伤害降低」各自独立成乘区，同一标签内线性叠加且 100% 封顶（与区间A对称）
  × ( 1 + R_受到伤害 )                                ← 区间C 通用槽：标签【受到伤害】，对所有伤害类型生效
  × ( 1 + R_受到X伤害 )                               ← 区间C 专属槽：【受到兵刃/谋略伤害】各自独立成乘区，按类型只命中其中之一
                                                       每个 R 槽内：免伤=负值、易伤=正值；
                                                       同方向乘法稀释：M_免伤 = 1 − Π(1 − mᵢ)，E_易伤 = 1 − Π(1 − eⱼ)；
                                                       异方向线性相减：R = E_易伤 − M_免伤（伤害乘区 = 1 + R）
  × ( 1 − 残兵减益 )                                   ← 攻方当前兵力不满带来的伤害衰减，满兵 = 0
  × 会心 / 奇谋伤害率                                  ← 触发时通常 ≥150%，未触发 = 1
  × 修正系数 (守方防御属性, 守方等级, 伤害类型)        ← 拟合常数项 ≈ 1，整体随守方防御呈二次函数式波动
```

- 主属性：兵刃伤害取攻方武力 vs 守方统率；谋略伤害取攻方智力 vs 守方智统 = (智力+统率)/2。
- 基础数值项有两种常见参数化：
  - article1 §9.1 给出的简化形式 `基础数值 360 + 攻方 − 守方`（兵刃 = 武力 − 统率；谋略 = 智力 − 智统/2，前面再加 360）。
  - 民间实测拟合形式 `固定值 ≈ 144 + 固定系数 ≈ 100 × (攻方 − 守方)/守方`（**非官方，实测来源待补**；仅作为对守方防御属性的边际递减解释参考。与简化形式在常见参数范围内数量级一致；若仅用于估算可直接采用 article1 简化形式即可）。


【区间A：攻方造成伤害类（线性 + 100% 封顶）】

- 涵盖：【造成伤害】【造成兵刃伤害】【造成谋略伤害】【主动战法伤害】【追击战法伤害】【普通攻击伤害】【对X类目标伤害】等所有「造成 XX 伤害」系数。
- 来源：科技增益 / 阵型增益 / 同兵种增益 / **兵种克制增益 (+15%，公开机制)** / 技能增减益 / 其他增减益（负面状态、缘分、装备等）。
- 同一标签（同一槽）内，所有来源**线性相加，无任何衰减**；100% 封顶。
- 不同标签（不同槽）在最终伤害公式里 **各自独立乘区**（多个 `(1+...)` 连乘形式）。其中【对X类目标伤害】下的每个子类——对武将 / 对统率低于自身 / 对带异常状态的敌军 等——也分别独立成乘区。

【残兵减益（独立尾乘项 `× (1 − 残兵减益)`）】

- 只取决于**攻方当前兵力**（与守方当前兵力无关）。
- 满兵 = 0 减益；以 10000 满兵为例，每减少 200 兵约 +1% 减益（8000 兵 ≈ 减 10%，6000 兵 ≈ 减 20%）。
- x（差兵数）与 y（减益百分比）疑似为边际递增的二次函数（非指数），偶尔出现「兵更少但伤害反而更高」即为函数局部走势所致。
- 这是"先手伤害很重要"的根本原因：开局先压低对方兵力 → 下个回合对方输出就被这一项扣掉；控制类武将（震慑/技穷）维持自身满兵也能间接提升输出。


【区间B：守方使敌方造成伤害降低（线性 + 100% 封顶）】

- 与区间 A 对称：守方的「使敌方……伤害降低」系数。
- 来源：科技增益 / 阵型增益 / 同兵种增益（PVE 时额外 ×0.92，article2 第 5 条）/ **兵种克制减伤 (−12%，隐藏机制，article2 第 4 条)** / 技能增减益 / 其他增减益（负面状态、缘分、装备等）。
  - **隐藏的 12% 克制减伤**：克制方攻击被克方时，被克方默认进入一段未在面板显示的 12% 减伤，落在区间 B 通用槽内。叠加面板上的克制 +15% 增伤后，单次克制净优势 ≈ 27%。
- 同一标签（同一槽）内**线性叠加，无任何衰减**；100% 封顶。不同标签之间各自独立成乘区（与区间 A 对称）。
- 守方伤害减益**与守方士兵数量无关**。
- **同百分点效益**：因区间 A、B 之间是乘法，**攻方想提升伤害**让对面减伤下降通常优于让自己增伤上升；**守方想降低伤害**让自己减伤上升通常优于让对面增伤下降。

【区间C：受到伤害类（同方向乘法稀释 + 异方向线性相减）】

- 涵盖：【受到伤害】【受到兵刃伤害】【受到谋略伤害】、以及【规避率】。
- 每个标签是**一个独立的 R 槽**（通用槽 R_通用 / 兵刃槽 R_兵刃 / 谋略槽 R_谋略），不同槽之间是**乘法相乘** (`(1+R_通用)·(1+R_X)`)；同一槽内的叠加规则分两层：
  - **同方向（同为免伤 或 同为易伤）按乘法稀释**：实际增加 = (1 − 当方向已有累积量) × 面板值，等价 `M_免伤 = 1 − Π(1 − mᵢ)`、`E_易伤 = 1 − Π(1 − eⱼ)`。
  - **异方向合并按线性相减**：净系数 `R = E_易伤 − M_免伤`，伤害乘区 = `1 + R`。
- 起点低于阈值（免伤 ~18%、易伤 ~20%）时近似线性，超过后衰减明显。
- **同方向单一大数值远比多个小数值划算**（乘法稀释的固有性质）。
- 经典反例：大乔[国色] 20% 基础易伤先发动后，再叠诸葛亮《出师表》12% 基础易伤只能增加 (1 − 20%) × 12% = 9.6%；若先 12% 再 20% 则两段都在阈值附近、几乎不衰减。
- 规避同理（同样是「同方向乘法稀释」规则；规避不参与与免伤/易伤的线性相减，单独走规避乘区）。

【区间 C 子节：免伤 ↔ 易伤 混合叠加示例】

  把累积「受到伤害」系数记作 R（带符号；R<0 净免伤，R>0 净易伤；初始 R=0），最终伤害倍率 = `1 + R`。混合叠加遵循两条规则：

  - **同方向多次叠加（同为免伤 或 同为易伤）按乘法稀释**：实际增加 = (1 − 当方向已有累积量) × 面板值。
  - **异方向合并按线性相减**：免伤桶 `M_免伤` 与易伤桶 `E_易伤` 各自先用乘法稀释叠完，净系数 `R = E_易伤 − M_免伤`。

  速算公式：
  ```
  M_免伤 = 1 − Π_i (1 − mᵢ)          ← 免伤序列乘法叠加
  E_易伤 = 1 − Π_j (1 − eⱼ)          ← 易伤序列乘法叠加
  R = E_易伤 − M_免伤
  最终伤害倍率 = 1 + R
  ```

  推演实例：

  - **场景 A（先免后易）**：先免 30%、再易 20% → R = 20% − 30% = **−10%**，倍率 = 0.90。
  - **场景 B（先易后免）**：先易 20%、再免 30% → R = 20% − 30% = **−10%**，倍率 = 0.90（与 A 同，异向顺序不敏感）。
  - **场景 C（双免再易）**：免 30%、再免 20%、再易 10%。
    - M = 1 − (1−0.30)(1−0.20) = 1 − 0.56 = **44%**；E = 10%。
    - R = 10% − 44% = **−34%**，倍率 = 0.66。
  - **场景 D（双易再免）**：易 20%、再易 15%、再免 30%。
    - E = 1 − (1−0.20)(1−0.15) = 1 − 0.68 = **32%**；M = 30%。
    - R = 32% − 30% = **+2%**，倍率 = 1.02。

  战略含义：

  - **同方向大数值远胜多个小数值**：场景 C 的两段免伤 30%+20% 名义 50%，乘法稀释后只剩 44%；若改成单一 50% 面板免伤，直接顶满。
  - **单段大易伤胜过多段小易伤**：易伤入桶后先做乘法稀释 `E = 1 − Π(1 − eⱼ)`，再与对方免伤桶**线性相减**。
    - 同样想拿 25% 净易伤：**单段 25% 面板易伤** → E = 25%；**五段 5% 面板易伤** → E = 1 − 0.95⁵ = **22.6%**，差 2.4 个百分点；段数越多稀释越严重。
    - 由于 E 与 M 之间是线性相减，每 1% 净易伤就直接抵 1% 净免伤（不被对方的免伤段数稀释）。所以**易伤的实战价值集中在"单段面板要大"**，而不是"段数多"。
    - 进一步：场景 D 中己方堆 32% 易伤，对方 30% 单段免伤就能把净 R 从 +32% 拉回 +2%；反之场景 C 中己方堆 44% 免伤，对方 10% 单段易伤可以把净 R 从 −44% 直接抬到 −34%（10 个百分点全数兑现）→ **己方在免伤桶里堆得越厚，对方一段不大不小的易伤反而更划算**。
  - **常见误区**：把易伤直接加到免伤系数上做线性算术（如「20+15−20−15=0」）。在 ≤2 段同方向时常常巧合相等，但**3 段及以上**误差变大，例如「20+15+10 易伤」E = 1 − 0.8·0.85·0.9 = **38.8%**（错误算法会得 45%）。

【区间 C 子节：多标签（多槽）的乘区叠加 —— 受到伤害 / 受到谋略伤害 / 受到兵刃伤害】

  「受到伤害」「受到谋略伤害」「受到兵刃伤害」是**三个不同的[特殊性质]标签**，对应**三个独立的 R 槽**，在最终伤害公式里**各自独立成乘区**。**只有同一标签内的免伤/易伤才走上一节的两层规则（同方向乘法稀释 + 异方向线性相减）**；不同标签之间则是 `(1 + R_i)` 各自成乘区，整体相乘。

  分槽规则：

  ```
  R_通用 = E_通用 − M_通用       ← 标签：【受到伤害】，对所有伤害类型生效
  R_谋略 = E_谋略 − M_谋略       ← 标签：【受到谋略伤害】，仅对谋略伤害生效
  R_兵刃 = E_兵刃 − M_兵刃       ← 标签：【受到兵刃伤害】，仅对兵刃伤害生效
  ```

  实际乘区拼装：

  - 谋略类技能：乘区部分 = `(1 + R_通用) × (1 + R_谋略)`
  - 兵刃类技能：乘区部分 = `(1 + R_通用) × (1 + R_兵刃)`
  - 普攻（视为兵刃）：走 `(1 + R_通用) × (1 + R_兵刃)`
  - 追击战法：按战法实际造成的伤害类型 (兵刃 / 谋略) 归属相应 R 槽 (R_兵刃 / R_谋略)；此外攻、守双方都有独立的【追击战法伤害】 / 【使敌方追击战法伤害降低】标签，分别落在区间 A 专属槽和区间 B 专属槽，各自成乘区。

  推演实例（守方挨一发谋略伤害）：

  守方身上有：【受到伤害】免伤 20% + 免伤 10% + 易伤 5%；【受到谋略伤害】免伤 15% + 免伤 12%。
  - Step 1（R_通用）：M = 1 − 0.8·0.9 = 28%；E = 5%；R_通用 = **−23%**。
  - Step 2（R_谋略）：M = 1 − 0.85·0.88 = 25.2%；E = 0；R_谋略 = **−25.2%**。
  - Step 3 合并：(1 − 0.23) × (1 − 0.252) = 0.77 × 0.748 = **0.5760**（受到的谋略伤害降为 57.6%，等效减伤 42.4%）。
  - ⚠️ 对比：若错误地把四段免伤 + 一段易伤都丢进一个槽，得到倍率 0.5886（减伤 41.14%）。真实「双乘区」算法**更强**，因为不同槽各自衰减得慢。

  战略含义：

  - 专属类型减伤价值高
  - **进攻镜像**：攻方的「使敌方受到 XX 伤害提升」专属易伤同样进入对应独立槽，比通用易伤更难被对面同槽免伤一次性抵消。

# 人工智能如何推荐演武武将 · 视频脚本

> 说明：所有数字均来自当前模型产物 `web/src/recommendation_data.json`
> （训练数据：2822 场战报，1793 个特征，回测准确率 73.6%）。
> 全片用「强度分数」的说法，不出现 sigmoid / 对数几率等术语；
> 「越高越可能赢」即可，保证通俗又不失准确。

## Scene 1 — 从战报到数据

**画面：**
- 展示截图 `image_extraction/fixtures/20260124-182348-609783_2e51719a.png`
- 高亮三块区域：① 武将头像/名字 ② 每个武将的战法 ③ 胜负结果
- 高亮区域依次「飞入」变成结构化数据 `…_2e51719a.json`
- 最后定格成 JSON 的列表形式（武将、战法、winner）

**文本（旁白）：**
> 今天我想讲清楚，人工智能到底是怎么帮我们推荐演武武将的。
> 听完这几分钟，你会对「AI 是怎么落地成一个演武工具」有个直观的认识。
>
> 一切从一张战报开始。算法拿到战报的第一步，不是「看懂」它，而是把里面有用的信息抽出来——
> 有哪些武将、每个武将带了什么战法、以及最关键的一条：谁赢了。
> 这样，一张图片就变成了一行干净的数据。

## Scene 2 — Feature Engineering：把武将变成 0 和 1

**画面：**
- 左边一支队伍（team 1）三个武将
- 右边逐行「点亮」特征列表，每行末尾是一个 0 或 1：
  ```
  H|姜维              : 1     （队里有姜维）
  HS|姜维|胜敌益强     : 1     （姜维带了胜敌益强）
  HP|姜维|诸葛亮       : 1     （姜维和诸葛亮同队）
  H|张三              : 0     （队里没有张三）
  …
  ```
- 镜头拉远：这样的特征有「几千种」（可打字幕：本模型 1793 个特征）

**文本（旁白）：**
> 但机器学习并不懂演武。它不知道「姜维」三个字是什么意思，它只认识 0 和 1。
> 所以我们要做一件事，叫「特征工程」。
>
> 思路很简单：把每一种可能的情况都变成一个「特征」。
> 比如——「队里有姜维」是一个特征；「姜维带了胜敌益强」是一个特征；「姜维和诸葛亮同队」又是一个特征。
> 有就是 1，没有就是 0。
>
> 就这样，把每个武将、武将和武将的搭配、武将和战法的搭配……所有可能的组合，
> 大概几千种，全都变成了一串 0 和 1。到这里，一支队伍就成了机器能读的语言。

---

## Scene 3 — 学习强度：算出每个特征的分数

**画面：**
- 展示核心公式（用「强度」而非「概率」的措辞）：
  ```
  队伍强度 = w1·H|姜维 + w2·HS|姜维|胜敌益强 + w3·HP|姜维|诸葛亮 + …
  谁的队伍强度高，谁就更可能赢
  ```
- 逐个高亮 w1、w2……并打问号「= ？」
- 动画：两边队伍都出现姜维 → 两个 +w 相互抵消 → 归零

**文本（旁白）：**
> 有了 0 和 1，我们就能写出这样一条式子：
> 把队伍里所有特征的分数加起来，得到这支队伍的「强度」。强度越高，就越可能获胜。
>
> 那每个特征前面的这个数字——w1、w2——从哪来？
> 这正是机器学习要做的事：从几千场战报里，反复计算，找出最合适的 w1、w2……
>
> 拿 w1 举例，它代表「队里有姜维，对获胜有多大帮助」。
> 如果带姜维的队伍普遍赢得多，这个数就会被算得比较高——它其实就代表了姜维的「强度」。
>
> 这里有个巧妙的地方：如果两边队伍都有姜维，一个 +、一个 −，正好抵消。
> 所以模型比的从来不是「绝对强」，而是「你比对手强多少」。

---

## Scene 4 — 结果与局限

**画面：**
- 网站截图：姜维的「强度加成」= **+0.2435**
- 对比出现一个负数例子（可用真实数据：孟获 单独强度 = **−0.5206**）
- 冷启动例子：文丑 强度加成 = **0.00**，旁边标注「只出场过 1 次 → 数据太少 → 被当成平均值」
- 结尾打字幕：训练 2822 场战报 · 回测准确率 73.6%（基准 52.3%）
- 引导评论区

**文本（旁白）：**
> 最后我们就得到了结果。比如姜维的强度加成是 +0.2435。
> 这个数可以是正的，也可以是负的——负的，说明这个武将在场，反而会拉低队伍胜率。
>
> 当然，这个算法远不完美。
> 最典型的一个坑是数据太少：有些武将根本上不了场，比如文丑，全部战报里只出现过一次。
> 数据不够，模型不敢下结论，干脆把它的强度记成 0，也就是「平均水平」——
> 但这并不代表文丑真的「中规中矩」，只是我们没见过它而已。
>
> 顺便说一句，这套模型是在 2822 场战报上训练的，回测准确率大约 73.6%，
> 比「瞎猜」的 52% 高出不少——说明它确实学到了东西。
>
> 你还能想到哪些缺陷？欢迎在评论区一起讨论。

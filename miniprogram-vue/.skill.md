# Game Advisor Mini Program — AI Visual Verification Skill

## Project Overview
- **Framework**: uni-app (Vue 3) + Wot Design Uni
- **Build targets**: H5 (web) for dev/testing, WeChat Mini Program for production
- **Dev server**: `npm run dev:h5` → `http://localhost:5173`
- **WeChat build**: `npm run build:mp-weixin` → `dist/build/mp-weixin/`

## Visual Verification Workflow

### 1. Start Dev Server
```bash
cd miniprogram-vue && npm run dev:h5
```

### 2. Take Screenshots with Playwright
```javascript
const { chromium } = require('playwright');
(async () => {
  const browser = await chromium.launch();
  const page = await browser.newPage({
    viewport: { width: 375, height: 812 },
    deviceScaleFactor: 2,
  });
  
  await page.goto('http://localhost:5173/', { waitUntil: 'networkidle' });
  await page.waitForTimeout(3000); // Wait for data to load
  await page.screenshot({ path: 'tests/tmp_rovodev_screenshot.png', fullPage: true });
  
  await browser.close();
})();
```

### 3. View Screenshots
Use `open_files` to view the PNG screenshot and inspect for visual issues.

### 4. Interactive Screenshots
```javascript
// Click to open picker, then screenshot
await page.locator('text=初始武将').click();
await page.waitForTimeout(1000);
await page.screenshot({ path: 'tests/tmp_rovodev_picker.png', fullPage: true });

// Type in search, then screenshot
await page.fill('input[placeholder*="搜索"]', 'cao');
await page.waitForTimeout(500);
await page.screenshot({ path: 'tests/tmp_rovodev_search.png', fullPage: true });
```

- `wd-loading` — loading state
- `wd-toast` — success notification

## Common Visual Issues & Fixes

### 1. Text/components too small or too large
- Design width is handled by uni-app automatically
- If Wot Design components look wrong, check `uni.scss` imports

### 2. Pinyin in picker labels
- Labels include capitalized pinyin: `曹操 Caocao`
- This matches mobile keyboard auto-capitalize behavior
- Filter is case-sensitive (uses `indexOf`)

### 3. Data loading errors
- H5 dev mode: data proxied through Vite (`vite.config.js` proxy)
- WeChat mode: fetched from Gitee (`https://gitee.com/liamqma/sanmou/raw/master/web/src/`)
- Gitee requires `Referer: https://gitee.com/` header (set in proxy config)

### 4. Sass deprecation warnings
- Pinned to `sass@1.77.8` to avoid `@import` deprecation warnings
- Do NOT upgrade sass beyond 1.77.x without checking for warnings

### 5. Wot Design overrides
- Use unscoped `<style>` (not `<style scoped>`) to pierce component boundaries
- Target classes like `.wd-card__content`, `.wd-cell__value`, etc.

## Verification Checklist
After any UI change, check:
- [ ] No error banners (red `wd-notice-bar`)
- [ ] Hero picker opens and shows filterable list
- [ ] Selected tags appear below picker cells
- [ ] Button is properly sized within card
- [ ] Text is readable (not too small/large)
- [ ] Spacing is consistent between sections
